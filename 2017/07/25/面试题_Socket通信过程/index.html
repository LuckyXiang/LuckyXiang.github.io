面试题 —— Socket通信过程

关于网络，首先大家要了解一下网络的基本内容。

网络如何通讯，首先要了解网络模型，当前最为广泛的模型有两种：OSI七层模型和TCP/IP四层模型。

网络通讯的三个重点要素：

  1. IP地址：网络设备的标识

  2. 端口号：用于标识进程的逻辑地址，有效端口：0–65535，其中0–1024为系统保留端口号

    一台拥有IP地址的主机可以提供许多服务，比如Web服务、FTP服务、SMTP服务等，这些服务完全可以通 过1个IP地址来实现。那么，主机是怎样区分不同的网络服务呢？显然不能只靠IP地址，因为IP 地址与 网络服务的关系是一对多的关系。实际上是通过“IP地址+端口号”来区分不同的服务的。

  3. 传输协议：数据传输必须遵循的规则，常见网络协议：TCP、UDP

Socket介绍

socket是对TCP、UDP协议的封装和应用，主要针对于程序员的(应用层面)，Socket本身并不是协议，而是一个调用接口(API)。通过Socket，我们才能使用TCP、UDP协议，简单的一个TCP/IP协议场景： 你要打电话给一个朋友，先拨号，朋友听到电话铃声后提起电话，这时你和你的朋友就建立起了连接，就可以讲话了。等交流结束，挂断电话结束此次交谈。

Socket套接字，用于描述IP地址和端口号，是支持TCP/IP、UDP协议的网络通信的基本操作单元，是面向于客户端/服务器(C/S)模型而设计的，解决客户端和服务器通讯的问题。

Socket基于TCP协议传输数据基本过程：

  1. 创建Socket：指定地址和端口号，分为ClientSocket、ServerSocket。
  2. Socket连接：客户端和服务器端建立连接。
  3. 打开Socket输入/输出流。
  4. 对Socket进行读/写操作。
  5. 关闭Socket。

案例实践

我们通过Socket建立一个基于TCP协议的聊天的通讯过程。

服务端

搭建服务端，我们简单的用一个java程序来搭建，运行开启即可。所以在AS中创建一个No Activity项目，建立一个类，创建main方法，在main方法中完成搭建，最后运行main方法即可。

1. 创建服务端socke

public static void main(String[] args) throws IOException {

    //建立ServerSocket,指定服务器的端口号
    ServerSocket ss = new ServerSocket(4700);

    //不断接收连接到此端口的socket
    // 并且为每一个socket开启一个线程，用于处理接收信息
    while (true) {

    // 获取连接到当前端口的所有socket
    Socket s = ss.accept();
    System.out.println("connect success!");

    // 为每一个Socket开启线程
    new Thread(new ServerThread(s)).start();
    }
}

我们在服务器端为每一个连接的Socket开新线程，处理信息，处理信息的内容，我们放置到一个特意创建的线程来处理接收的消息及返回消息给客户端。

public class ServerThread implements Runnable {

    private Socket mSocket = null;
    private BufferedReader mBufferedReader = null;

    // 构造函数中接收socket，并且初始化BufferedReader，用于读取客户端发送的消息
    public ServerThread(Socket socket) throws IOException {
        mSocket = socket;
        mBufferedReader = new BufferedReader(
                new InputStreamReader(mSocket.getInputStream(), "utf-8"));
    }

    @Override
    public void run() {
        try {
            String content = null;
            //循环接收来自此客户端的消息
            while ((content = mBufferedReader.readLine()) != null) {
                System.out.println(content);
                try {
                    // 拿到连接的客户端socket的输出流，用于数据输出给客户端
                    OutputStream os = mSocket.getOutputStream();

                    // 当客户端发送bye，结束通话
                    if (content.equals("bye")) {
                        os.write(("通话结束").getBytes("utf-8"));

                        // 结束之后关闭流关闭客户端Socket
                        os.close();
                        mSocket.close();
                        return;
                    }
                    // 向客户端回应消息
                    os.write((content + ":来自服务器的返回。\n").getByte("utf-8"));
                } catch (SocketException e) {
                    e.printStackTrace();
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Socket通信过程

总结过程：

服务器端：

  1. 创建一个ServerSocket，用于监听客户端Socket的连接请求
  2. 采用循环不断接受来自客户端的请求
  3. 每当接受到客户端Socket的请求，服务器端也对应产生一个Socket客户端
  4. IO流读写操作进行数据的传递
  5. 关闭socket

客户端：

  1. 创建客户端socket(指定服务器端IP地址与端口号)
  2. 连接(Android 创建Socket时会自动连接)
  3. 与服务器端进行通信:IO流读写的操作
  4. 关闭socket

附：其他常问面试题

TCP与UDP的区别？

UDP:
将数据源和目的封装成数据包中，不需要建立连接
每个数据包的大小在限制在 64k
因无连接，是不可靠协议
不需要建立连接，速度快

TCP:
通过三次握手完成连接，是可靠协议
建立连接，形成传输数据的通道
在连接中进行大数据量传输
必须建立连接，效率会稍低
