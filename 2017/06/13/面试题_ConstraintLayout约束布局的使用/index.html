面试题 —— ConstraintLayout的使用

ConstraintLayout的介绍

ConstraintLayout允许您使用平面视图层次结构创建大而复杂的布局（无嵌套视图组）。 它类似于RelativeLayout，因为所有View都依赖于兄弟控件和父控件之间的相对关系，但它比RelativeLayout更灵活，并且更易于与Android Studio的布局编辑器一起使用。

ConstraintLayout的所有功能都可以直接从布局编辑器的可视化工具中获得，因为布局API和布局编辑器是专门为彼此构建的。 因此，您可以使用ConstraintLayout完全通过拖拽而不是编辑XML来构建布局。此外，ConstraintLayout可以兼容到Android 2.3(API9),所以不用担心兼容问题。

想要在ConstraintLayout中定义一个View的位置，你至少需要添加两个约束：一个水平方向，一个竖直方向。当然，你可以添加更多的约束。
如果你没有指定某个方向上的约束，那么该View将会在坐标(0,0)或（x，0）或（0，y）处显示（这取决于你没有指定哪个方向的约束）。
如下图，View-C没有指定垂直方向的约束，虽然在预览时你看到的效果没有问题，但是运行后会发现，该View的显示坐标为（x，0）。

ConstraintLayout的使用

使用之前确保你的项目已经添加了ConstraintLayout的包。

  1. 点击Tools–> Android–> SDK Manager.
  2. 点击SDK Tools的选项.
  3. 打开Support Repository 然后下载ConstraintLayout for Android 和 Solver for ConstraintLayout ，然后点击OK下载即可.

  4. 添加依赖库到build.gradle里面，同步即可。

    compile ‘com.android.support.constraint:constraint-layout:1.0.2’

创建ConstraintLayout布局

将一个已存在的布局转换为ConstraintLayout布局。

  1. 打开Android studio 的布局 –> Design
  2. 在Component Tree窗口，右键–> Convert layout to ConstraintLayout.

添加布局约束

当在ConstraintLayout中添加视图时，它会在每个角上显示一个带有方块调整大小句柄的边框，并在每一边显示圆形约束句柄。

点击视图，然后单击并按住其中一个约束句柄并将该行拖动到可用的锚点（另一个视图的边缘，布局的边缘或指南）。

创建约束时，请记住以下规则：

  • 每个视图必须至少有两个约束：一个水平和一个垂直。
  • 您只能在约束句柄和共享相同平面的锚点之间创建约束。 因此，视图的垂直平面（左侧和右侧）只能被约束到另一个垂直平面; 基线只能限制到其他基线。
  • 每个约束句柄只能用于一个约束，但您可以创建多个约束（从不同的视图）到相同的锚点。

1. Parent position：

视图约束到布局的相应边缘。

2. Order position：

定义两个视图的垂直或水平的外观位置。

3. Alignment

基线对齐：视图文本对准另一个视图文本基线。

删除约束

添加过约束之后，如果需要删除：

1. 删除控件单个约束条件：

将鼠标悬浮在某个约束的圆圈上，然后该圆圈会变成红色，这个时候单击一下就能删除了。

2. 删除控件所有约束：

删除某一个控件的所有约束，选中一个控件，然后它的左下角会出现一个删除约束的图标，点击该图标就能删除当前控件的所有约束了。

3. 删除当前布局所有约束：

删除当前界面中的所有约束，点击工具栏中的删除约束图标即可。

自动添加约束

如果界面中的内容变得复杂起来，给每个控件一个个地添加约束也是一件很繁琐的事情。为此，ConstraintLayout中支持自动添加约束的功能，可以极大程度上简化那些繁琐的操作。

自动添加约束的方式主要有两种：一种叫Autoconnect，一种叫Inference。

Autoconnect：

默认情况下是关闭的，使用前先打开，在布局中添加一个视图，自动生成约束。

Autoconnect会判断我们的意图，并自动给控件添加约束。不过Autoconnect是无法保证百分百准确判断出我们的意图的，如果自动添加的约束并不是你想要的话，还可以在任何时候进行手动修改。总之，可以把它当成一个辅助工具，但不能完全靠它去添加控件的约束。

Inference：

Inference也是用于自动添加约束的，但它比Autoconnect的功能要更为强大，因为AutoConnect只能给当前操作的控件自动添加约束，而Inference会给当前界面中的所有元素自动添加约束。因而Inference比较适合用来实现复杂度比较高的界面，它可以一键自动生成所有的约束。
